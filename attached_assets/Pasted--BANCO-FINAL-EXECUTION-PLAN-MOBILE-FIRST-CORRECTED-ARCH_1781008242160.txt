# 🚀 BANCO — FINAL EXECUTION PLAN (MOBILE-FIRST CORRECTED ARCHITECTURE)

**ROLE:** Act as a Principal System Architect and Mobile-First Tech Lead.
**🛑 CRITICAL CORRECTION:** The Consumer Marketplace is a **MOBILE APPLICATION** (React Native / Expo). It is NOT a web app. The Dealer OS is a **B2B WEB DASHBOARD** (Next.js / React).

You are NOT building features blindly. You are building 3 STRICTLY ISOLATED, connected systems.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
## 0) SYSTEM SPLIT (CRITICAL & NON-NEGOTIABLE)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
BANCO = 
1. **CORE ENGINE:** (Shared Backend API + DB)
2. **CONSUMER MARKETPLACE:** (Mobile Application - React Native/Expo)
3. **DEALER OS:** (B2B Web SaaS Dashboard - Next.js)

❗ NOTHING crosses layers directly. Consumer Mobile App MUST NOT contain Dealer logic.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
## 1. CORE ENGINE (BUILD FIRST)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
*   **Tech:** Node.js, Drizzle ORM, PostgreSQL.
*   **Contains:** Database schema, Ranking engine, Payment normalization logic, Ads engine, Lead tracking, Search parser.
*   **Public APIs:** `/feed`, `/listings/:id`, `/search`, `/leads`
*   **Private APIs:** `/dealer/*`
*   **Rule:** NO UI code exists in this layer. Pure JSON APIs.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
## 2. CONSUMER MARKETPLACE (MOBILE APP - AFTER CORE)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
*   **Tech:** React Native (Expo), Framer Motion (for Mobile), React Navigation.
*   **Consumes:** `/feed`, `/listings`, `/search`
*   **Features:** Infinite mobile feed, swipeable media galleries, sticky bottom CTAs (Call/WhatsApp), Save listings.
*   **Rule:** Optimized for one-hand mobile usage. Haptic feedback on actions. NO dealer/inventory management logic here.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
## 3. DEALER OS (WEB DASHBOARD - AFTER CORE)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
*   **Tech:** Next.js / Vite (Web), Tailwind CSS, DataGrid components.
*   **Consumes:** `/dealer/listings`, `/dealer/leads`, `/dealer/ads`, `/dealer/bulk-import`
*   **Features:** Inventory management, Bulk CSV import pipeline, CRM lead table, Analytics charts.
*   **Rule:** Desktop-optimized. High data density.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
## 4. DATABASE & API LAYER RULES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
*   **Single Database:** Consumer reads, Dealer writes, Core Engine validates.
*   **API Middleware:** Strict role enforcement. The API Gateway must block any Consumer token from accessing `/dealer/*` routes to ensure complete Tenant Isolation.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
