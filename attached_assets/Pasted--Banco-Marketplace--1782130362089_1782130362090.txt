نظام الصلاحيات — Banco Marketplace

ملاحظة تنفيذية

ارجو العمل باقل تكلفة واتمام المهام بشكل صحيح بدون الرجوع اليها مرة اخري

⸻

1) تعريف الأدوار (Roles)

type Role = 'owner' | 'admin' | 'moderator' | 'support' | 'user'

⸻

2) صلاحيات كل دور

👑 OWNER (المالك)

* تعيين أو إزالة Admin / Moderator / Support
* تغيير أي Role لأي مستخدم
* Ban / Unban لأي مستخدم
* إدارة كل الإعدادات
* الوصول الكامل لكل البيانات

قيود:

* لا يمكن حذفه
* لا يمكن تغيير دوره
* يجب وجود Owner واحد دائمًا

⸻

🛡️ ADMIN (مدير)

* Ban / Unban المستخدمين
* إدارة الإعلانات (حذف / إيقاف)
* إدارة الأقسام (Categories)
* إدارة البلاغات (Reports)
* مراجعة المحتوى

قيود:

* لا يمكنه تعيين Admin آخر
* لا يمكنه تغيير Owner
* لا يمكنه تعديل Roles

⸻

🔍 MODERATOR (مشرف)

* مراجعة البلاغات
* إخفاء / إيقاف الإعلانات المخالفة
* Ban / Unban (مؤقت)

قيود:

* لا يمكنه إدارة Roles
* لا يمكنه الوصول للإعدادات

⸻

🎧 SUPPORT (خدمة عملاء)

* قراءة بيانات المستخدمين
* مشاهدة الطلبات
* الرد على العملاء

قيود:

* لا يمكنه حذف إعلانات
* لا يمكنه Ban / Unban
* لا يمكنه تغيير Roles

⸻

👤 USER (مستخدم عادي)

* نشر إعلان
* تعديل إعلانه
* حذف إعلانه
* مراسلة المستخدمين
* الشراء والبيع

⸻

3) حالات المستخدم (User Status)

type UserStatus = 'active' | 'suspended' | 'banned' | 'deleted'

⸻

4) التحقق (Verification)

isVerified: boolean

⸻

5) القواعد الأمنية (Guards)

* لا يمكن حذف آخر Owner
* لا يمكن لأي مستخدم تعديل نفسه (role)
* لا يمكن لـ Admin أو Moderator تعديل Roles
* Owner فقط هو من يدير الصلاحيات
* كل العمليات الحساسة تسجل في Audit Log

⸻

6) سجل العمليات (Audit Log)

{
  actorId: string,
  targetUserId: string,
  action: 'SET_ROLE' | 'BAN' | 'UNBAN' | 'DELETE_AD',
  oldValue: any,
  newValue: any,
  createdAt: Date
}

⸻

7) API

تغيير Role

PATCH /admin/users/:id/role
body: {
  role: 'admin' | 'moderator' | 'support' | 'user'
}

⸻

Ban / Unban

PATCH /admin/users/:id/ban

⸻

8) UI (صفحة Users)

لكل مستخدم:

* عرض البيانات
* عرض الإعلانات
* زر Ban / Unban
* زر Verify / Unverify
* Dropdown لتغيير Role

⸻

9) قواعد التنفيذ

* كل الصلاحيات تنفذ في Backend فقط
* Frontend للعرض فقط
* استخدام authGuard للتحقق من الدور
* أي عملية بدون تحقق = مرفوضة
