#36 - Dealer OS Upgrades
What & Why
The dealer web portal can manage and analyze existing inventory but cannot actually create or edit a listing from the UI, has no way to respond to RFQs, and shows no billing/ROI. To be launch-ready for sellers, Dealer OS needs: a full create/edit listing form (including financing offers, geo/location, taxonomy, and industrial specs), an RFQ inbox where dealers respond with structured offers, a wallet/billing view (CPL deductions and balance), tier/subscription visibility, and ROI on the dashboard. This consumes the financing/offers, industrial, profile, and RFQ data from the backend tasks and the wallet/billing data from the existing monetization task. It lives entirely in the dealer-os artifact, so it can proceed in parallel with the mobile tasks.

Done looks like
A dealer can create and edit a listing through a form that captures category-appropriate taxonomy, location/geo, financing offers (Islamic and/or Conventional), and industrial specs where relevant.
A dealer sees an RFQ inbox and can respond to a request with a structured offer (price, monthly, provider).
A wallet/billing view shows the balance and CPL deduction history.
Subscription tier and its limits/benefits are visible.
The dashboard shows ROI (lead/ad spend vs. results) alongside existing KPIs.
All views use real BFF data and the app typechecks.
Out of scope
Backend data model, financing computation, RFQ, and wallet endpoints (consumed from the Payments Engine, B2B Backend, and existing monetization tasks).
Admin god-mode / moderation (covered by the existing Admin control center task).
Mobile app changes.
Steps
Create/Edit listing UI — Build a create and edit listing form covering taxonomy, location/geo, financing offers (dual-mode), and industrial specs, wired to the existing listing create/update endpoints.
RFQ inbox — Add an inbox listing incoming RFQs with the ability to respond using a structured offer.
Wallet/billing view — Add a page showing wallet balance and CPL deduction history.
Tiers + ROI — Surface the dealer's subscription tier/limits and add an ROI panel to the dashboard.
