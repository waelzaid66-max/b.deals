# 🚀 BANCO V11: FINANCIAL ENGINE & OPS MASTER
**ROLE:** Principal Fintech Architect & Product Operations Lead.
**MISSION:** Build the "Iron-Clad" financial engine, billing system, and the Admin Control Room (The Dashboard).

**🛑 CORE DIRECTIVE:**
The system must guarantee that every "WhatsApp Click" = "Revenue". Every "Boost" = "Wallet Deduction". Any discrepancy in the wallet balance is a catastrophic failure.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
## 1. THE FINANCIAL INTEGRITY ENGINE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
* **Atomic Transactions:** Every action that deducts money (Boost) or charges money (Lead) MUST be wrapped in a PostgreSQL Transaction.
    * *Example:* If a Lead is generated, the system must simultaneously:
        1. Insert into `lead_billing` (Pending).
        2. Update `wallet_balance` (Deduction).
        3. Insert into `transactions_log`.
    * If one fails, the entire action must rollback.
* **Audit Trail:** Every change in `wallet.balance` MUST be linked to a `transaction_id`. Never update a balance without a linked log entry.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
## 2. THE SAAS & PAYMENTS LOGIC
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
* **Subscription Enforcement:** If `subscription.end_date < NOW()`, the `listings` created by that dealer must automatically switch status to 'archived'. 
* **The "Lead" Ledger:** Since you are charging 5-7 EGP per lead, the system must prevent "Click Bombing" (Fake Clicks).
    * *Anti-Abuse:* Ignore multiple clicks from the same IP/Device on the same listing within a 60-second window.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
## 3. ADMIN CONTROL CENTER (THE "GOD MODE")
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
* **Real-Time Revenue Aggregation:** A `GET /api/admin/revenue` endpoint that returns:
    * Total Revenue (MTD - Month to Date).
    * Revenue by Channel (Subscriptions vs. Boosts vs. Leads).
* **Moderation Queue:** Any listing flagged by users or categorized as "High Risk" (Industrial/Large Real Estate) must appear in the Admin Dashboard with a "Pending Approval" status.
* **The Support "Escalation" Flow:**
    * Support tickets (`support_tickets`) must show "Time Since Last Reply". If > 24 hours, highlight in Red for the Admin.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
## 4. SYSTEM STABILITY & ANALYTICS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
* **Business KPI Dashboard:**
    * *Conversion Rate:* (Leads / Total Views).
    * *Profitability per User:* (Revenue Generated - Operational Cost).
    * *Trending Assets:* Which category is driving the most profit?
* **Alerts Pipeline:** If `transaction_status == 'failed'` for more than 5 times in an hour, trigger an alert to the Admin Telegram/Email.
