# 🚀 BANCO MOBILE: PRODUCTION-READY UX & PERFORMANCE ENGINE
**ROLE:** Principal Mobile Architect (React Native / Expo).
**MISSION:** Deliver a high-performance, conversion-focused mobile experience.

**🛑 CORE PRODUCTION RULES:**
* **Zero UI Logic:** The app is a "dumb" rendering layer. It receives data from the BFF and renders it. No formatting, no math.
* **Performance:** Use `FlashList` (from Shopify) instead of standard `FlatList` for 10x faster feed rendering.
* **UX Integrity:** Never interrupt the user. All network calls (lead tracking, behavior signals) must happen in the background (Non-blocking).

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
## 1. THE INSTANT STARTUP (Dark Launch)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
* **Splash Screen:** Force `#000000` background. Use `expo-splash-screen` to keep the screen visible until the first feed data is cached or ready.
* **User Interface Style:** Force `userInterfaceStyle: "dark"` in `app.json`.
* **Zero-White-Flash:** Ensure the native root view has a black background color defined in the native manifest.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
## 2. THE HIGH-CONVERSION MOBILE UX
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
* **Conversion Priority:** 
    * Cards must follow the **[Visual] -> [Price] -> [Installment]** hierarchy strictly.
    * Use `react-native-reanimated` for the "Tap-to-Scale" (0.96) effect. It must feel physical and responsive.
* **Native Action System:**
    * Use `Linking.openURL()` for WhatsApp/Call. 
    * Trigger the API `POST /leads/track` *before* opening the URL. If the API fails, the user still gets their WhatsApp/Call (User experience is king).
* **Deep Linking:** Implement `Linking` config in Expo to route `banco://listing/:id` directly to the `ListingDetailScreen`.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
## 3. PERFORMANCE & MEMORY OPTIMIZATION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
* **Media Handling:** Use `expo-image` (it handles caching and blurring images automatically). 
* **Component Memoization:** Use `React.memo` on the `SmartAssetCard` to prevent re-rendering the whole feed when only one card is interacted with.
* **Image Optimization:** Ensure the server provides different thumbnail sizes (e.g., 300px width for feed, 800px for details).

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
## 4. REAL-WORLD "CONVERSION" FEATURES (The Missing Logic)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
* **Pull-to-Refresh:** Essential. Must trigger a full refresh of the feed and clear the memory cache.
* **Offline Fallback:** If the API fails (network timeout), show a "Retry" button that is easy to reach. Do not show an empty white screen.
* **Image Preloading:** Preload images of the next 5 items in the FlatList to make the scroll feel "bottomless".
* **Interaction Haptics:** Use `expo-haptics` to vibrate slightly when a user taps "WhatsApp" or "Call". It confirms action to the user physically.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
## EXECUTION COMMAND:
1. Initialize the `App.tsx` with the black-background splash configuration.
2. Implement the `ConversionAssetCard` using `FlashList` for high performance.
3. Integrate `expo-haptics` for all lead-generating actions.
