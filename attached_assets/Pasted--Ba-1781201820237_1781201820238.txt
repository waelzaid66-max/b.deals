المطلوب الآن إعادة ضبط التنفيذ بالكامل بناءً على الهدف الحقيقي للمشروع.

الوضع الحالي:
تم بناء Backend + Admin + Dealer Systems بشكل كبير، لكن هذه الأنظمة غير مكتملة الظهور داخل تطبيق الموبايل، وبالتالي المنتج النهائي غير مكتمل.

المشكلة ليست في البيانات أو الـ APIs، بل في عدم تحويل هذه الأنظمة إلى تجربة مستخدم كاملة داخل التطبيق.

لذلك، المطلوب فورًا:

━━━━━━━━━━━━━━━━━━━━━━━

1. إيقاف كامل لأي تطوير Backend / Admin / Monetization

لا يتم تنفيذ أي Task جديدة خارج الموبايل.

أي Feature لا تظهر داخل الموبايل تعتبر غير موجودة.

━━━━━━━━━━━━━━━━━━━━━━━

2. تحويل كل ما تم بناؤه إلى واجهة مستخدم كاملة داخل الموبايل

ابدأ فورًا بربط:

* Listings → تظهر ببيانات حقيقية داخل الـ Home Feed
* Dealer Data → يظهر داخل الكارت والتفاصيل
* Payment Options → تظهر داخل صفحة التفاصيل
* Leads → يتم تفعيلها عند الضغط على الأزرار
* RFQ → يظهر ويعمل كـ Flow كامل وليس زر فقط

━━━━━━━━━━━━━━━━━━━━━━━

3. استكمال SmartAssetCard (Core Conversion Component)

يجب أن يحتوي كل كارت على:

* صورة / فيديو
* السعر الكاش بشكل واضح وكبير
* أقل قسط شهري (Lowest Monthly)
* Badge:
    Verified / Trending / Installment
* الموقع
* مؤشر تفاعل (views / contacted)

الكارت يجب أن يبيع، ليس فقط يعرض

━━━━━━━━━━━━━━━━━━━━━━━

4. استكمال Listing Detail (أهم شاشة في النظام)

يجب أن تحتوي على:

A) السعر:

* Cash Price (واضح جدًا)
* “Starts from X/month”

B) نظام التقسيط:

* Islamic Plan (ثابت)
* Bank Installment (down payment + monthly)
* عرض أفضل عرض (Best Offer)
* عرض باقي العروض

C) التاجر:

* الاسم
* النوع (Dealer / Company)
* حالة التوثيق

D) أزرار التحويل (Conversion):

* Call
* WhatsApp
* Request Offer (Flow كامل)

E) المواصفات:

* ديناميك حسب النوع (سيارات / عقارات / صناعي)

━━━━━━━━━━━━━━━━━━━━━━━

5. استكمال Home Feed (ليس List عادية)

يجب تقسيمه إلى Sections واضحة:

* Best Deals
* Installment Opportunities
* Verified Sellers
* Near You
* High Demand
* Recently Added

كل Section له منطق عرض مختلف

━━━━━━━━━━━━━━━━━━━━━━━

6. استكمال User Flow (End-to-End)

يجب أن يعمل السيناريو التالي بدون أي نقص:

* فتح التطبيق
* رؤية listings حقيقية
* الضغط على Listing
* فهم السعر والتقسيط فورًا
* اختيار التواصل أو Request Offer

أي خطوة ناقصة = فشل في التنفيذ

━━━━━━━━━━━━━━━━━━━━━━━

7. تعريف “Feature Done”

أي Feature لا تعتبر مكتملة إلا إذا:

✔ تظهر داخل الموبايل
✔ مفهومة بصريًا
✔ قابلة للاستخدام
✔ متصلة بباقي النظام

غير ذلك = غير منتهية

━━━━━━━━━━━━━━━━━━━━━━━

8. ممنوع أي تشتت

* لا Screenshots لإثبات إن البيانات موجودة
* لا Logs analysis بدون سبب
* لا Features جديدة

التركيز فقط على:

→ إكمال ما تم بناؤه
→ ربطه داخل الموبايل
→ تحويله لتجربة كاملة

━━━━━━━━━━━━━━━━━━━━━━━

الهدف النهائي:

تحويل التطبيق من:

Demo يعرض بيانات

إلى:

Product حقيقي يحقق Conversion

━━━━━━━━━━━━━━━━━━━━━━━

ابدأ فورًا بتنفيذ:

Feed → Detail → Payment → Contact → RFQ
